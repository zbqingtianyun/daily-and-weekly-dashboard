## 项目概览

这是一个日报/周报经营数据看板：

- 前端：Next.js 15、React 19、TypeScript、ECharts。
- 数据源：MySQL 数据库 `edu_company` 中的 `日报`、`周报`表。
- 数据链路：`scripts/sync_data.py` 只读查询数据库，生成 `data/*.json` 静态快照，前端读取快照展示。
- 测试：Vitest 单元测试与 Playwright 端到端测试。
- 包管理器：固定使用 `pnpm@10.12.1`，不要混用 npm 或 yarn，也不要生成其他锁文件。

## 目录职责

- `app/`：Next.js App Router 页面、布局、加载态、错误态和全局样式。
- `components/`：仪表盘与图表组件。
- `lib/`：指标计算、格式化、类型定义等共享逻辑。
- `data/`：可部署的日报、周报、指标目录与元数据快照。
- `scripts/`：数据同步、指标目录和离线数据校验脚本。
- `database/`：MySQL 连接与查询工具；本地凭据仅放在 `database/.env`。
- `tests/`：Vitest 单元测试；`tests/e2e/` 存放 Playwright 测试。

## 开发约定

- TypeScript 保持严格模式，优先复用 `lib/types.ts` 中的类型；项目内导入可使用 `@/` 别名。
- 指标计算、格式化和成熟窗口逻辑集中在 `lib/metrics.ts`，不要在组件中重复实现。
- 指标定义以 `scripts/metric_catalog.py` 为源头；字段变更必须同步检查生成的指标目录、类型、展示逻辑和测试。
- 保持日报按自然日、周报按源表 `当周周一` 的现有口径。
- 次日、7 日、14 日留存必须遵守成熟窗口：未成熟数据展示为空或“待成熟”，不得把尾部零值当作真实下降。
- 不假设业务漏斗阶段严格包含；优先使用源表已有转化率或现有指标逻辑。
- UI 改动应兼顾桌面端和移动端，并保留加载态、错误态及无数据状态。
- 除非任务明确要求，不手工批量编辑生成的数据快照；应通过同步脚本更新。

## 数据库与数据安全

- `pnpm data:sync` 的标准用途是只读同步 `日报` 和 `周报`，不得擅自改成写库操作。
- 禁止提交 `database/.env`、任何 `.env`、查询结果目录或真实凭据。
- 使用 `database/query_db.py` 时默认保持只读；任何 `--allow-write`、DDL、删除、清空、权限或结构变更都必须先获得用户明确授权，并说明目标数据库、SQL、影响范围及回滚/备份方案。
- 展示查询结果时避免输出不必要的敏感字段和大批量原始数据。

## 常用命令

```powershell
corepack enable
pnpm install
pnpm dev
pnpm typecheck
pnpm test
pnpm test:e2e
pnpm data:check
pnpm build
```

需要数据库凭据时才运行：

```powershell
pnpm data:sync
```

## 验证要求

根据改动范围执行最小但充分的验证：

- 指标计算或类型变更：`pnpm typecheck`、`pnpm test`。
- 数据脚本或快照变更：`pnpm data:check`；若已配置本地数据库，再运行 `pnpm data:sync`。
- 页面、组件或样式变更：`pnpm typecheck`、`pnpm test`、`pnpm build`；交互或响应式行为受影响时补充或运行 `pnpm test:e2e`。
- 配置或跨模块改动：至少执行 `pnpm build`。

测试应覆盖本次行为变化，尤其关注无效分母、空值、百分比格式、周期排序、留存成熟窗口及日报/周报切换。不要通过删除或放宽既有测试来掩盖问题。
